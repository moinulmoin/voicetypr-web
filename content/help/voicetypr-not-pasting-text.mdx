## What this usually means

If Voicetypr transcribes but nothing appears in the app you are using, the issue is usually not the microphone or model.

It is usually one of these:

- Accessibility permission is missing on macOS
- the target app is elevated on Windows and Voicetypr is not
- the target field is not focused
- the hotkey or paste step is blocked by another app

## macOS fix

1. Open **System Settings → Privacy & Security → Accessibility**
2. Make sure **Voicetypr** is enabled
3. If you just enabled it, quit and reopen Voicetypr
4. Focus a simple text field and test again

## Windows fix

1. Click into a simple app like Notepad
2. Test Voicetypr there first
3. If it works in Notepad but not another app, try running Voicetypr as administrator
4. Test again in the app that was failing

This matters when the target app is running with higher privileges.

## Check the target field

Before dictating, make sure:

- your cursor is actually inside a text field
- the app is active and focused
- a modal or dialog is not stealing focus

## Check the hotkey flow

If dictation starts but no paste happens consistently:

- try a different hotkey
- close apps that also capture global shortcuts
- test in one plain app before moving back to Cursor, Slack, or your browser

## Best quick test

Use this order:

1. Notepad or TextEdit
2. Notes / Apple Notes
3. Gmail or a browser textarea
4. Cursor or another complex app

That tells you whether the problem is global or app-specific.
