## What this warning means

macOS Gatekeeper blocks apps that are not yet notarized or broadly recognized by Apple.

It does **not** automatically mean the app is unsafe.
It means macOS wants you to explicitly allow it.

## Fix it

1. Move VoiceTypr into **Applications**
2. Try opening it once from Applications
3. When macOS blocks it, close the warning
4. Open **System Settings → Privacy & Security**
5. Scroll to the Security section near the bottom
6. Click **Open Anyway** next to VoiceTypr
7. Confirm with your password or Touch ID
8. Launch VoiceTypr again
9. Click **Open** on the final confirmation

## If Open Anyway does not appear

Try this:

1. Close System Settings
2. Launch VoiceTypr once again from Applications
3. Re-open **Privacy & Security**
4. Wait a few seconds for the button to appear

If it still does not show, reboot once and repeat the steps.

## If the app opens once and then disappears

That usually means:

- the app was run from the disk image instead of Applications
- permission changes were made while the app was still running

Fix:

1. Quit VoiceTypr fully
2. Confirm the app is inside **Applications**
3. Launch it again from Applications
