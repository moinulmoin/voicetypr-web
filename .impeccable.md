# VoiceTypr — Design Context

This file is the durable design brief for VoiceTypr's web surface
(voicetypr.com). It is the source of truth for typography, color, voice,
layout rhythm, and the things we deliberately are not. When in doubt about
a design decision, consult this file first; it wins over training-data
defaults.

## Users

Solo founders, indie hackers, AI power users, and developers — people
whose day is mostly typing English at AI tools (Cursor, Claude, ChatGPT)
rather than writing code. They use VoiceTypr 8–12 hours a day, on macOS
or Windows, alongside the rest of their working environment. They have
strong taste, low patience for marketing language, and have already tried
at least one cloud dictation tool before landing here.

The job: replace the 80–150 wpm typing bottleneck with ~150 wpm voice,
without giving up privacy, without paying a subscription, without
disrupting the apps they already use.

## Brand Personality

Three words: **warm, mechanical, opinionated.**

- **Warm** — built by a single person who answers his own support email.
  Voice should feel like a founder talking, not a brand talking.
- **Mechanical** — the product is precise, technical, and uncompromising
  about local-first / privacy. The interface should feel like an
  instrument, not a service.
- **Opinionated** — pay-once-not-subscription is a contrarian position
  that needs to be defended visually, not apologized for.

### Voice rules

- First person ("I built this") over third person ("VoiceTypr is").
- Concrete numbers over adjectives: "~5× faster than typing", not "much
  faster".
- Acknowledge tradeoffs honestly. Tools, not movements.
- No "transform / unleash / dive in / game-changing" language.
- No emoji in marketing copy or UI.
- Em-dashes sparingly — kept for headlines and ledes, not body prose.

## Aesthetic Direction

**Technical-warm.** Sans-dominant with a serif italic appearing as a
**rare punctuation move**, not as the default headline font. We are a
developer tool with a soul, not a magazine that happens to sell software.

- Light mode first. **Pure white** background (`#ffffff`). Surface-2
  sections (Pricing, Outcomes, tonal breaks) use a subtle warm gray
  (`#f5f5f4`) so those moments still register as a different beat
  without the rest of the site feeling cream/precious.
- One accent color: deep green `#3a7a4f`. Used sparingly — accents work
  *because* they are rare.
- Two web fonts: **Geist** (everything default — body, eyebrow strips,
  meta, labels, **and headings h1–h5** at weight 700 + tight tracking)
  + **Instrument Serif** (loaded italic-only-ish, weights 400; reserved
  for the rare `<em>` punctuation accent inside hero / brand-moment H2 /
  final CTA, plus opt-in editorial numerals via Tailwind's `font-serif`
  utility). Mono falls back to the system stack
  (`ui-monospace, "SF Mono", Menlo, Consolas, monospace`); no download.
- Asymmetric layouts where they earn their place. Not every section is a
  3-card grid.
- Motion: Apple-curve easing (`cubic-bezier(0.32, 0.72, 0, 1)`), 300ms
  baseline. `active:scale-[0.98]` on every tactile button.

### References (the right vibe)

- Linear's restraint
- MKBHD store's typographic confidence
- Substack's reading experience
- Karri Saarinen's personal site
- Hardcover.app's warmth

### Anti-references (do **not** look like this)

- Generic AI/SaaS landing pages with purple-pink gradients,
  glassmorphism, stock illustrations
- Dark mode with neon accents and "AI cyan" palettes
- Subscription-product visual language: pricing-toggle drama, "Most
  popular" defensive ribbons, FOMO timers
- Magazine pastiche where the serif does all the work
- Founder personality cosplay — "made by humans" badges, hand-drawn
  arrows, intentionally messy fonts
- Open-source / GitHub badges on the front face. Footer-tier signal,
  never a hero claim. We are not positioning on open-source as a value
  prop in 2026.

## Design Principles

1. **Confident, not precious.** A founder built this for himself. Pages
   should look certain about what they are, not coy.
2. **One signature move per page, not five.** Italic-serif accents,
   button-in-button arrow sockets, dashed meta strips — choose one as
   the dominant detail per page; let the others rest.
3. **Numbers over adjectives.** Outcomes panels show `5×`, `0`, `∞` —
   not "10× more productive." If a metric isn't real, don't fake one.
4. **Local-first as visual language, not as a slogan.** The design itself
   should feel installed-on-your-machine: sharp edges, real letterforms,
   small file sizes, no cloud-tooling chrome.
5. **Italic em as rare punctuation.** The serif italic accent appears on
   page heroes, brand-moment H2s (Pricing "Yours forever", Compare "for
   less?"), and final CTAs. It does **not** appear on every internal
   section header. If everything is emphasized, nothing is.
6. **GitHub / open-source visible only to people who go looking for it.**
   Footer-tier signal in the trust column. Never above-the-fold copy.

## Typography Tokens

| Role | Font | Weights / Styles |
|---|---|---|
| Sans (body, eyebrow, meta, labels, **headings h1–h5**) | Geist | variable, all weights; headings use 700 |
| Italic em accent (rare punctuation move only) + opt-in editorial numerals | Instrument Serif | 400 normal + 400 italic |
| Mono (code blocks only) | System stack — no download | — |

- Fluid `clamp()` only on hero / page-title H1s. Fixed `rem` on body,
  cards, dashboards, changelog, blog post bodies.
- Italic-em accent used in ≤ ~14 places site-wide. Reuse the previous
  rules in `globals.css` (`.landing-editorial em`) — italic, accent
  color, serif family.

## Layout Rhythm

- `.ed-section` — 88px / 56px (mobile) vertical padding. Use for every
  marketing section.
- `.ed-container` — max-width 1240px, 40px / 20px gutters.
- `.ed-eyebrow` — small-caps tracked label with green accent dot. Once
  per section, never twice.
- Card pattern repetition: avoid the same 3-card grid more than twice in
  the scroll. Mix in asymmetric layouts (single-card features,
  side-by-side compare, full-bleed visual moments).
