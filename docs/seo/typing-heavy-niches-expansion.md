## Core conclusion

VoiceTypr is no longer just a Windows/accessibility SEO play.

The product is a strong fit for a much broader market: people whose work is bottlenecked by typing long-form text in normal desktop apps. That includes people writing prompts, docs, replies, notes, specs, case notes, pitches, and updates all day.

The next SEO wave should lean into **typing-heavy professions** and **workflow-heavy knowledge work**, not just accessibility pain states.

## Positioning frame to keep consistent

Use the same design structure and the same product truth:

- dictation-first, not full voice control
- local transcription by default
- works in the apps people already use
- pay once, not another SaaS subscription
- useful when typing is slow, painful, tiring, or simply too expensive in time

Do not drift into claims that imply:

- meeting bot / team transcript software
- medical treatment
- full computer control
- compliance guarantees that are not explicitly supported

## Copy rules for profession pages

Each page should answer these five things clearly:
1. What typing-heavy work this person actually does.
2. Why typing becomes the bottleneck.
3. Where VoiceTypr fits in their real tool stack.
4. Why local-first / pay-once / every-app input matters for them.
5. What VoiceTypr is not for.

### Examples by page
- Journalists: not for multi-speaker newsroom transcription.
- Product managers: not a meeting bot or PRD generator.
- Customer support: not full CRM automation.
- Lawyers: not a legal compliance guarantee.
- Students: not a study app or plagiarism shortcut.

- per-page `article:modified_time` / `og:updated_time` metadata
- page-level `FAQPage` JSON-LD generated from the existing FAQ data
- in-body contextual links between adjacent use cases inside the workflows section

The GSC typo/pricing pass also strengthens `/voicetyper` for visible early queries such as `voicetyper`, `voice typer app`, `voicetyper pricing`, and `voicetyper cost`. The page now has pricing-aware metadata, visible FAQ copy, FAQ schema, and a homepage FAQ link without changing the main homepage layout.

The key rule still stands: harden the existing cluster before adding another batch of pages.

## What is already good to go

- The Windows cluster is directionally solid.
- The accessibility cluster is directionally solid.

- Related guides + footer guide links already give the cluster crawl paths without bloating the homepage.

## What is not complete yet

- VoiceTypr still over-indexes on Windows/accessibility relative to the bigger “people who type all day” market, but the gap is much smaller now.
- Profession coverage is now thirteen pages deep: developers, writers, founders, journalists, product managers, customer support, lawyers, students, researchers, recruiters, sales, marketers, and consultants.
- The next expansion should be less about obvious professions and more about adjacent workflow pages, stronger in-body links, freshness signals, and page-level schema.
- The footer guide set now reflects the broader market without turning the site into a directory.

## Recommended rollout order after this pass

1. Ship, request indexing, and watch which profession pages start getting impressions first.
2. Add workflow pages that cut across professions, like client recaps, AI prompting, long-form email, and notes-to-draft.
3. Add cleaner “best for / not for” blocks only to the pages that start getting impressions.
4. Add sourced stats only where they improve trust and can be defended.

## Final recommendation

Yes, push harder — but do it through a **profession/use-case cluster expansion**, not by bloating the homepage and not by publishing generic “best dictation software” fluff.

VoiceTypr’s sharpest SEO wedge is:

**“voice typing for people who write a lot, across real desktop apps, with local transcription by default and pay-once pricing.”**

That is bigger than Windows.
That is bigger than accessibility.
And it is much more ownable than trying to win generic speech-to-text terms head-on.
